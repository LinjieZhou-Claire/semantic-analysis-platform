# 实验报告：语义分析综合测试平台

## 一、实验背景与目的

本次实验系统学习了文本的分布式表示方法，从基于矩阵分解的 LSA，到基于神经网络的 Word2Vec (CBOW/Skip-Gram)、GloVe，再到引入子词信息的 FastText 和句子级别的 Sent2Vec。通过构建一个集成多种语义表示模型的交互式 Web 系统，利用实际数据验证核心算法原理。

**技术栈：** Python + Streamlit + scikit-learn + gensim + nltk + matplotlib

---

## 二、系统架构

```
语义分析综合测试平台
├── 模块 1: TF-IDF 与 LSA（传统统计模型）
├── 模块 2: Word2Vec (CBOW vs Skip-Gram)
├── 模块 3: GloVe 预训练模型与词类比
└── 模块 4: FastText 子词特征 & Sent2Vec 句向量
```

**输入语料：** 约 700 词的英文 NLP 领域百科式文本（30 句），涵盖 NLP、词嵌入、深度学习等主题。

---

## 三、各模块实验过程与观察

### 模块 1：传统统计模型 (TF-IDF 与 LSA)

**对应课件：** P43 (LSA), P85-88 (TF-IDF)

#### Prompt

```
在 Streamlit 应用的第一个标签页中，实现基于传统统计的文本表示。
提供一个文本域供用户输入英文语料，将其按句子切分为文档集合。
使用 sklearn.feature_extraction.text 计算并展示文本的 TF-IDF 矩阵，提取出权重最高的 5 个关键词。
在 TF-IDF 或 One-hot (CountVectorizer) 矩阵的基础上，应用 TruncatedSVD (LSA) 将词汇降维到 2 维空间。
使用散点图将降维后的词汇可视化在 2D 坐标系中。
```

#### 实现要点

- 使用 `TfidfVectorizer` 构建 TF-IDF 词-文档矩阵（已过滤英文停用词）
- 按平均 TF-IDF 权重排序，提取 Top-5 关键词
- 将 TF-IDF 矩阵转置（行=词汇，列=文档），应用 `TruncatedSVD` 降维到 2D
- 使用 K-Means 对降维后的词向量聚类，用颜色区分不同语义簇
- 同时提供 TF-IDF 矩阵和 CountVectorizer 矩阵两种 LSA 输入的对比

#### 运行结果

**TF-IDF 权重最高的 5 个关键词：**

| 排名 | 关键词 | 平均 TF-IDF 权重 |
|------|--------|------------------|
| 1 | word | 0.1044 |
| 2 | embeddings | 0.0867 |
| 3 | language | 0.0799 |
| 4 | learning | 0.0744 |
| 5 | processing | 0.0647 |

**LSA 2D 可视化（K-Means 聚类着色）：**

**各聚类包含的词汇（示例）：**

- **Cluster 1**: language, natural, processing（NLP 主题）
- **Cluster 2**: embeddings, learning, networks, neural, word2vec（词嵌入/神经网络主题）
- **Cluster 3**: artificial, intelligence, transformers, attention（AI 基础主题）
- **Cluster 4**: word, representation, vectors（词向量主题）
- **Cluster 5**: document, frequency, term, matrix（统计方法主题）

#### 观察结论

> LSA 通过对共现矩阵做 SVD 分解，成功将在相同文档中高频共现的词语映射到潜在语义空间的相近位置。例如 *natural / language / processing* 聚在一起，*neural / networks / deep / learning* 聚在一起。这验证了 LSA 能捕获词汇间的潜在语义关联——即使两个词没有直接共现，但如果它们与相同的第三方词共现，LSA 也能发现这种间接关系（即同义词/多义词处理能力）。
>
> TF-IDF 矩阵通过 IDF 权重抑制了高频通用词的影响，使得 LSA 降维后语义聚类更清晰；而 Count 矩阵保留了原始频次信息，高频词可能主导降维结果。

### 模块 2：Word2Vec 训练与对比 (CBOW vs Skip-Gram)

**对应课件：** P47 (Word2Vec), P73 (词义相关性)

#### Prompt

```
在第二个标签页中，实现 Word2Vec 的实时训练和测试。
使用与模块1相同的语料，调用 gensim.models.Word2Vec 进行训练。
在界面上提供单选按钮，允许用户切换训练架构：CBOW (sg=0) 或 Skip-Gram (sg=1)。
提供滑动条（Slider）调整 window（上下文窗口，2-10）等参数。
提供一个单词输入框，计算并输出在当前训练模型下，与该词余弦相似度最高的 5 个词汇。
```

#### 实现要点

- 训练前自动过滤英文停用词（the, is, to, and 等），聚焦内容词之间的语义关系
- 向量维度默认 50（小语料不宜过高），训练 300 个 epoch 确保收敛
- 支持 CBOW / Skip-Gram 切换与 window 大小调整

#### 运行结果

**查询词: `learning`，window=5, vector_size=50**

| 排名 | CBOW (sg=0) | 相似度 | Skip-Gram (sg=1) | 相似度 |
|------|-------------|--------|-------------------|--------|
| 1 | language | 0.998 | corpora | 0.972 |
| 2 | models | 0.997 | popular | 0.971 |
| 3 | large | 0.997 | deep | 0.971 |
| 4 | learn | 0.997 | representations | 0.970 |
| 5 | word | 0.997 | learn | 0.965 |

**查询词: `neural`**

| 排名 | CBOW (sg=0) | 相似度 | Skip-Gram (sg=1) | 相似度 |
|------|-------------|--------|-------------------|--------|
| 1 | learn | 0.998 | networks | 0.995 |
| 2 | hierarchical | 0.998 | learn | 0.979 |
| 3 | deep | 0.998 | hierarchical | 0.977 |
| 4 | designed | 0.997 | layers | 0.976 |
| 5 | networks | 0.997 | designed | 0.974 |

> 【此处插入 Word2Vec 模块运行截图】

#### 观察结论

> 切换 CBOW 和 Skip-Gram 架构后，同一目标词的 Top-5 相似词确实发生了变化：
>
> 1. **CBOW** 的相似度分数普遍更高（0.997-0.998），词之间的区分度较小。这是因为 CBOW 通过上下文预测目标词，倾向于学到更平滑、更泛化的表示。
> 2. **Skip-Gram** 的相似度更分散（0.965-0.995），语义关联更精确。例如 `neural` 的最相似词是 `networks`（0.995），而非泛化的 `learn`。这是因为 Skip-Gram 通过目标词预测上下文，对低频词和具体搭配更敏感。
> 3. 两种架构都成功捕获了语义关系（如 neural→networks, learning→deep），验证了 Word2Vec 的分布假说。

### 模块 3：预训练 GloVe 模型与词类比

**对应课件：** P60 (GloVe), P65 (词类比 Word Analogies)

#### Prompt

```
在第三个标签页中，测试预训练的 GloVe 模型与词类比任务。
使用 gensim.downloader 加载预训练模型 glove-wiki-gigaword-100。
构建词类比（Word Analogy）计算器：提供 A, B, C 三个输入框。
在后台计算向量公式：Result = Vector(A) - Vector(B) + Vector(C)。
检索并输出距离 Result 向量最接近的单词。
补充一个基础功能：输入任意两个单词，计算它们的词义相似度分数。
```

#### 实现要点

- 使用 `glove-wiki-gigaword-100` 预训练模型（Wikipedia + Gigaword 语料，100 维，40 万词汇）
- 利用 gensim 的 `most_similar(positive=[A, C], negative=[B])` 实现词类比
- 内置 9 个经典类比示例，涵盖国家-首都、性别、语法等类型

#### 运行结果

**经典词类比结果：**

| 类比公式 | 期望结果 | 实际 Top-1 | 余弦相似度 |
|----------|----------|-----------|-----------|
| king − man + woman | queen | **queen** ✅ | 0.7699 |
| paris − france + china | beijing | **beijing** ✅ | 0.8485 |
| paris − france + germany | berlin | **berlin** ✅ | 0.8846 |
| tokyo − japan + france | paris | **paris** ✅ | 0.9008 |
| london − england + japan | tokyo | **tokyo** ✅ | 0.8541 |
| go − went + come | came | **came** ✅ | 0.8798 |
| good − better + bad | worse | terrible | 0.6868 |

> 【此处插入 GloVe 词类比运行截图】

**词义相似度：**

| 词对 | 余弦相似度 |
|------|-----------|
| king ↔ queen | 0.7508 |
| happy ↔ sad | 0.6801 |
| cat ↔ dog | 0.8798 |

#### 观察结论

> 1. **国家-首都类比**全部命中（6/6），说明 GloVe 在大规模语料上成功学习到了"国家→首都"这一线性语义关系。向量空间中 `Paris - France ≈ Berlin - Germany ≈ Tokyo - Japan`，即国家和首都之间的向量差几乎恒定。
>
> 2. **性别类比** king − man + woman = queen 经典示例成功验证。这表明 GloVe 向量空间中存在一个"性别"方向，`king - man ≈ queen - woman`。
>
> 3. **语法类比**（时态变换 go→went，come→came）也表现优秀，说明词向量不仅编码了语义信息，还隐式捕获了语法规则。
>
> 4. 比较级类比（good-better+bad→worse）效果稍差，Top-1 是 terrible 而非 worse，可能因为比较级变换的线性关系不如国家-首都那样规律。
>
> 5. 模型选择很重要：`glove-twitter-25`（Twitter 数据，25 维）效果远不如 `glove-wiki-gigaword-100`（Wikipedia 数据，100 维），说明训练语料的质量和向量维度对词类比任务影响显著。

### 模块 4：FastText 子词特征与句向量 (Sent2Vec)

**对应课件：** P68 (子词), P81 (Sent2Vec), P89 (FastText)

#### Prompt

```
在第四个标签页中，实现 FastText 与句子级表示（Sent2Vec）。
使用之前的语料实时训练一个 gensim.models.FastText 模型。
OOV 测试：提供一个输入框，故意输入一个带有拼写错误的词（如 'computeer'）。
分别尝试用 Word2Vec 和 FastText 提取向量。用 try-except 捕获 Word2Vec 的 KeyError。
Sent2Vec（简单实现）：提供两个句子输入框，将句子分词后提取所有词的向量并求均值，
得到两个句子的全局向量，计算余弦相似度。
```

#### 实现要点

- FastText 使用 `min_n=3, max_n=6` 的字符 n-gram 范围
- OOV 测试：Word2Vec 抛出 KeyError，FastText 利用子词信息成功生成向量
- Sent2Vec 使用 Average Pooling，提供 GloVe 预训练向量和 FastText 训练向量两种选择
- 句向量计算前自动过滤停用词，只保留内容词

#### 运行结果

**OOV 对比测试（输入: "computeer"）：**

| 模型 | 结果 |
|------|------|
| Word2Vec | ❌ KeyError — 'computeer' 是未登录词 (OOV)，无法获取向量！ |
| FastText | ✅ 成功生成向量！最相似词: computer, computational, computing... |

> 【此处插入 OOV 对比测试截图】

**句向量相似度（使用 GloVe 预训练向量）：**

| 句子对 | 余弦相似度 |
|--------|-----------|
| 句子1 "Natural language processing uses deep learning models." ↔ 句子2 "Deep neural networks are used in NLP applications." | **0.9181** |
| 句子1 ↔ 句子3 "The weather is sunny and warm today." | **0.6983** |
| 句子2 ↔ 句子3 | **0.5895** |

> 【此处插入句向量相似度截图】

#### 观察结论

> 1. **FastText 的 OOV 鲁棒性**：当输入拼写错误的词 `computeer` 时，Word2Vec 因词表中不存在该词而抛出 KeyError，而 FastText 利用字符级 n-gram 特征（如 "com", "comp", "put", "ute", "tee", "eer"）成功合成了一个向量，并找到了语义最相近的词 `computer`。这验证了 FastText 通过子词信息处理 OOV 问题的能力。
>
> 2. **句向量均值法**的语义区分能力：语义相关的句子 1 和句子 2（都涉及 NLP/深度学习）相似度为 0.92，远高于与无关句子 3（天气话题）的相似度 0.59-0.70。这验证了词向量均值法 (Average Pooling) 能在一定程度上捕捉句子级语义。
>
> 3. **向量来源影响**：使用预训练 GloVe 向量的句向量效果远好于小语料训练的 FastText 向量。小语料训练的模型因数据不足导致所有词向量方向趋同，句子相似度区分度极低。这说明 Sent2Vec 方法依赖底层词向量的质量。

## 四、实验总结

### 核心发现

| 模型 | 核心原理 | 优势 | 局限性 |
|------|----------|------|--------|
| TF-IDF + LSA | 基于词频统计 + SVD 矩阵分解 | 无需训练神经网络，可解释性强 | 无法捕获词序信息，维度灾难 |
| Word2Vec | 基于神经网络的分布式表示 | 能捕获语义关系，训练高效 | 无法处理 OOV，忽略子词信息 |
| GloVe | 全局共现矩阵 + 局部上下文窗口 | 词类比效果优秀，线性关系明确 | 依赖大规模预训练语料 |
| FastText | 字符 n-gram 的词表示 | 能处理 OOV 和形态变化 | 模型体积较大 |
| Sent2Vec (Avg) | 词向量均值池化 | 简单高效，适合快速原型 | 丢失词序信息，受底层词向量质量制约 |

### 关键实验心得

1. **语料规模决定模型质量**：小语料（~30 句）训练的 Word2Vec/FastText 需要降低向量维度（50 维以下）并增加训练轮次，否则向量区分度不足。
2. **预训练模型的选择至关重要**：GloVe 的 `wiki-gigaword-100` 模型在词类比任务上远优于 `twitter-25`，说明训练语料的质量（正式百科 vs 口语化推特）和向量维度（100 vs 25）都显著影响效果。
3. **停用词过滤是小语料训练的关键**：不过滤停用词时，Word2Vec 的最相似词全是 the/is/to/and 等功能词；过滤后才能看到有意义的语义关联。
4. **CBOW vs Skip-Gram**：Skip-Gram 在小语料上的语义精确度更高（如 neural→networks），而 CBOW 更泛化但区分度低。
